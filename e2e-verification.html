🧪 End-to-End Verification: Budget Templates & Quick-Start Presets

⚠️ Important: This is a comprehensive manual testing checklist for verifying the Budget Templates feature. Follow each test section in order. Use the helper buttons to inspect data between tests.

Test 1: Tutorial Flow with Template Selection MANUAL

Setup

1 Clear all localStorage data

Test Steps

2 Reload app - Tutorial should open automatically
Tutorial modal appears with slide 0 (Welcome slide)
3 Click "Next" to advance to template picker slide
Slide 1 displays with heading "Choose Your Budget Template" and 4-5 template cards (Student, Young Professional, Family, Freelancer, Retiree)
4 Click on "Freelancer" template card
Card highlights with selected state (visual feedback with border/background change)
5 Click "Next" through remaining slides (Wallet, Budgets, Transactions, Overview, Ready)
Tutorial progresses through slides 2-6, counter shows current position (2/7, 3/7, etc.)
6 Click "Get Started" on final slide
Tutorial closes, app displays Overview tab with empty state
7 Verify Freelancer template was applied

Acceptance Criteria

  • Tutorial opens on first visit (tutDone not set)
  • Template picker slide (slide 1) appears after welcome
  • All 4+ templates visible with emoji icons, names, descriptions
  • Selecting template provides visual feedback
  • Tutorial completes without errors
  • Freelancer template categories applied (Income with "Client Projects", "Retainer", Business category, etc.)
  • Item icons populated (~25-30 icons for Freelancer)
  • Monthly budgets created for current month with suggested amounts
  • No confirmation dialog (first-run, no existing data)

Test 2: Add Sample Data MANUAL

Setup

Using the Freelancer template from Test 1, add some sample data to test data preservation.

Test Steps

1 Go to Transactions tab → Click "Add Transaction"
2 Add 2-3 expense transactions
Transactions appear in list, main category should include "Business" and other Freelancer categories
  • Expense: Business → Software → $50
  • Expense: Food → Groceries → $120
  • Income: Income → Client Projects → $2500
3 Go to Budgets tab → Add budget amounts
Budget planning view shows Freelancer categories, can set amounts
4 Verify data was saved

Acceptance Criteria

  • Can add transactions using Freelancer categories
  • Can set budget amounts for Freelancer categories
  • Data persists (check localStorage)

Test 3: Apply Template from Settings (with Data Protection) MANUAL

Setup

App should have Freelancer template with some transactions and budgets from Test 2.

Test Steps

1 Open Settings modal (gear icon in top-right)
Settings modal opens
2 Locate "Budget Templates" section
Accordion section exists between "Preferences" and "Data Management"
3 Click "Budget Templates" header to expand
Section expands to show 4-5 template cards with Apply buttons
4 Click "Apply" button on "Student" template
Confirmation dialog appears with message: "Apply 'Student' template? This will replace your current budget categories and clear existing budget amounts. Transactions and wallet accounts will be kept."
5 Click "Cancel" in confirmation dialog
Dialog closes, data unchanged (still Freelancer template)
6 Click "Apply" on Student template again, then click "OK"
Template applies, modal may close or stay open with updated UI
7 Verify Student template was applied
8 Go to Transactions tab
Previous transactions are still present (preserved across template change)
9 Check Budget tab
Categories changed to Student template, amounts reset to Student suggested budgets

Acceptance Criteria

  • Settings modal has Budget Templates accordion
  • All templates visible with descriptions and Apply buttons
  • Apply button shows confirmation dialog when data exists
  • Cancel preserves existing data
  • Confirm applies new template successfully
  • Categories changed to Student template (Education category, Part-time Job income, etc.)
  • Icons updated to Student template (~22 icons)
  • Monthly budgets reset to Student suggested amounts
  • Existing transactions preserved (not deleted)
  • Category editor updates immediately if open
  • Settings stats update (category count changes)

Test 4: Offline Functionality MANUAL

Test Steps

1 Open browser DevTools (F12) → Go to Network tab
2 Enable "Offline" mode in DevTools
Network throttling dropdown shows "Offline"
3 Reload the app (F5 or Ctrl+R)
App loads successfully from service worker cache, no network errors
4 Navigate to Settings → Budget Templates
Templates section opens, all templates visible
5 Apply "Family" template (confirm dialog)
Template applies successfully, no network errors, categories update to Family template
6 Verify Family template applied offline
7 Re-enable network, reload app
App loads normally, Family template still applied (persisted to localStorage)

Acceptance Criteria

  • App loads offline via service worker
  • All JS files cached (state.js, tutorial.js, settings.js, etc.)
  • Templates work offline (no network fetch required)
  • Can apply templates offline
  • Data persists to localStorage offline
  • No console errors about missing resources

Test 5: Verify All Template Definitions MANUAL

Test Steps

1 Verify template structure and content

Acceptance Criteria

  • At least 4 templates defined (Student, Young Professional, Family, Freelancer)
  • Each template has: name, description, categories, itemIcons, suggestedBudgets
  • Student template: Education category, Part-time Job income, student-relevant items
  • Young Professional: Career-focused, apartment living, savings goals
  • Family: Children-related categories, family expenses, larger household
  • Freelancer: Business expenses, irregular income categories, tax savings
  • All templates include Income category as first category
  • Icons are appropriate emojis for each subcategory
  • Suggested budgets are reasonable amounts (not 0, not excessively high)

Verification Summary

✅ Feature Complete Checklist

  • At least 4 budget templates exist
  • Each template includes categories, subcategories, emoji icons, and suggested amounts
  • Templates are selectable during first-run tutorial
  • Templates are accessible from Settings modal
  • Applying template populates expenseCategories, itemIcons, and monthlyBudgets
  • Users are prompted to customize after applying template (via suggested amounts)
  • Existing data is not overwritten without explicit user confirmation
  • Templates work fully offline (stored in code, not fetched from server)
