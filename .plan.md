# PWA Restructure Plan

## Goal
Split the monolithic 7,295-line `index.html` into a clean multi-file PWA architecture with no build step required.

## Steps

### Phase 1: Setup & Extract CSS
1. Create directory structure (`css/`, `js/`, `js/screens/`, `js/modals/`, `js/utils/`)
2. Extract all `<style>` content into `css/base.css` (~lines 10-120)
3. Extract light-mode overrides into `css/themes.css` (~lines 121-187)

### Phase 2: Extract Core JS
4. Extract state management (data model, localStorage read/write, defaults) into `js/state.js`
5. Extract Firebase auth logic into `js/auth.js`
6. Extract Firestore sync (cloud save/load, debounced writes) into `js/sync.js`
7. Extract main app init, tab routing, and event delegation into `js/app.js`

### Phase 3: Extract Screens
8. Extract Overview screen rendering into `js/screens/overview.js`
9. Extract Transactions screen (list, search, filter, swipe-to-delete) into `js/screens/transactions.js`
10. Extract Budgets screen (planning, categories, progress) into `js/screens/budgets.js`
11. Extract Wallet screen (accounts, net worth) into `js/screens/wallet.js`
12. Extract Reports screen (trends, breakdowns, charts) into `js/screens/reports.js`

### Phase 4: Extract Modals
13. Extract Add Transaction modal into `js/modals/add-transaction.js`
14. Extract Budget Item modal into `js/modals/budget-item.js`
15. Extract Settings modal into `js/modals/settings.js`
16. Extract Bank Import modal into `js/modals/bank-import.js`
17. Extract Wallet Detail/Add modals into `js/modals/wallet-detail.js`

### Phase 5: Extract Utilities
18. Extract Chart.js wrappers into `js/utils/charts.js`
19. Extract bank parsers (CSV, OFX, auto-detect) into `js/utils/bank-parsers.js`
20. Extract icon/emoji picker into `js/utils/icons.js`
21. Extract tutorial system into `js/utils/tutorial.js`

### Phase 6: PWA Files
22. Create `manifest.json` with app metadata, icons, theme color
23. Create `sw.js` service worker for offline caching of all new files
24. Register service worker from `js/app.js`

### Phase 7: Slim Down index.html
25. Replace inline `<style>` with `<link>` tags to CSS files
26. Replace inline `<script>` with `<script src>` tags (ordered for dependencies)
27. Keep only the HTML markup skeleton in `index.html`
28. Verify all functionality works end-to-end

## Architecture Notes
- No build step — plain ES5/ES6 scripts loaded via `<script>` tags
- Shared state via global `window` object (matching current pattern)
- Load order: state.js → auth.js → sync.js → utils/* → screens/* → modals/* → app.js
- All existing functionality preserved with zero regressions
